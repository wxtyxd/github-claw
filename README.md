# github-claw 🦀

> 一个持久化的 GitHub Copilot 个人工作空间——让 AI 助手跨对话继承角色、记忆和工作方式。

---

## 📌 这是什么？

**github-claw** 是受 OpenClaw 启发的个人 AI 助手工作空间。它将 GitHub 仓库作为持久化的"大脑"，让 GitHub Copilot 能够：

- 🧠 **跨对话保持一致的角色**：每次新对话都能自动继承工作方式和记忆
- 📚 **积累长期知识**：将学到的模式、决策、代码片段永久保存
- 🔄 **系统化工作流程**：从任务启动、执行、收尾都有清晰的规范
- 🎯 **可扩展且轻量**：文件即记忆，Markdown 即规范，易于理解和维护

---

## 🚀 快速开始

### 对于新的 Copilot 对话

在启动新对话时，**复制以下 prompt**：

```markdown
我在 GitHub 仓库 wxtyxd/github-claw 中维护了一份工作手册。
请先查阅它来了解我期望的工作方式：

1. 查看 AGENTS.md - 了解你的角色、工作原则和收尾流程
2. 查看 PROGRESS.md - 了解当前任务状态
3. 按其中定义的方式继续工作

开始前确认你已准备好。
```

然后，Copilot 会自动：
- 查阅核心文档
- 继承角色和工作方式
- 保持工作的连贯性

### 对于贡献者

如果你想为这个工作空间做出贡献或适配到你自己的项目：

1. **理解框架**：查看 [AGENTS.md](./AGENTS.md)
2. **检查日志**：查看 [memory/daily/](./memory/daily/) 了解进展
3. **复用模式**：参考 [memory/patterns.md](./memory/patterns.md) 中的工作模式

## 💡 核心特性

### 1. 持久化记忆系统
```
长期记忆（1+年保留）
├── AGENTS.md - 角色和规范
├── memory/context.md - 项目决策
└── memory/patterns.md - 工作模式

进度追踪（1-2月保留）
├── PROGRESS.md - 任务状态
└── memory/daily/ - 日志档案
```

### 2. 工作流程完整性
- ✅ **启动**：快速上下文加载
- ✅ **执行**：系统化任务管理
- ✅ **收尾**：知识沉淀和文件更新
- ✅ **继承**：下个对话自动接棒

### 3. 轻量且可扩展
- 📝 Markdown 作为记忆载体
- 🔍 易于人类阅读和 AI 理解
- 📈 支持随时添加新的模式和决策
- 🎯 反对过度设计，优先实用性
